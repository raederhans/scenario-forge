前线形状改进方向

如何让前线看起来更像真实的军事态势线？

当前：沿行政边界
目标：平滑军事态势线

改进方案

A

自动曲线平滑

原理：在现有 mesh path 基础上，用 Catmull-Rom 或 B-spline 对路径进行自动平滑。保留 controller 数据驱动，前线仍从 topology mesh 生成，但渲染时自动将锐角和折线柔化为曲线。

优势

  • 改动最小，完全兼容现有数据
  • 自动化，无需用户手动操作
  • 可调平滑程度

局限

  • 仍受限于 feature 粒度
  • 平滑后可能偏离真实行政边界
  • 视觉改善但底层问题未解决
B

手绘前线工具

原理：新增前线绘制工具（类似 Operation Graphics 的绘制方式），用户直接在地图上手绘前线路径。前线不再由 controller 数据生成，而是用户自由绘制的路径。可以沿河流、山脉走向画出自然的前线。

优势

  • 完全自由，精确表达任何态势
  • 可沿地理特征走向
  • 最大灵活性

局限

  • 需要用户手动画每条前线
  • 与 controller 数据脱耦
  • 开发量较大
C

混合方案：自动生成 + 手动微调 (推荐)

原理：前线仍由 controller 数据自动生成（保证数据一致性），但生成后自动平滑，并允许用户拖拽控制点来微调路径。用户可以在自动生成的前线上添加/移动控制点，让它沿河流或山脉走向弯曲。

优势

  • 兼顾数据驱动和视觉精度
  • 默认自动生成，减少手动工作
  • 保留 controller 数据的一致性
  • 渐进式增强，可以先做平滑再加控制点

局限

  • 实现复杂度中等
  • 需要管理额外的控制点数据
