1

地块上的定位混乱

兵牌放在 feature 上时，要么布置在中心（堆叠时重合），要么偏向随机角度。当前代码按 featureId 或经纬度近似值分组堆叠，但初始定位依赖点击位置或 feature centroid。没有智能布局逻辑。

2

视觉效果 — 缺乏图标

当前有两种渲染模式："game" = 只显示文字代码 (INF/ARM/ART...)，"milstd" = 军标符号（SIDC）。但 milstd 图标生成依赖 SIDC 码，普通用户难以使用。缺少简单直观的图标选择器。

3

国家间差别不大

不同国家的兵牌唯一视觉区分只有左侧的窄色条（nation strip, 宽度≈12%）和顶部 tag pill 中的 3 字母国家代码。在缩放较远时几乎无法区分德国和苏联的兵牌。

4

UI 缩略图表达

侧边栏中的兵牌列表/编辑器 UI 信息密度高。预览卡片有完整的组织度、装备、编队等信息，但在列表视图中显得拥挤。需要更好的缩略 vs 详情分级。

5

整体 UI 过于拥挤

兵牌编辑器、前线控制、Operation Graphics 都在侧边栏中，空间紧张。可以考虑弹出面板、分 Tab、或点击兵牌后弹出详情窗口来缓解。
