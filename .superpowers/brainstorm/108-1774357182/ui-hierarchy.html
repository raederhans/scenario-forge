UI 分级与拥挤问题

当前侧边栏承载了太多功能，需要重新组织信息层级

当前 UI 结构问题

现状：左侧边栏（296px 宽）
▸ Scenario 场景控制
Mode selector / Apply / Reset
▸ Inspector 检查器
Frontline overlay toggle + style
▸ Strategic Overlay
Operation Graphics editor
── 绘制模式 / 样式 / 控制点
Unit Counter editor ← 占用大量空间
── Preset / Nation / Echelon
── Preview card
── Detail drawer (展开时更大)
   ── Renderer / Size / SubLabel
   ── Symbol code / SIDC
   ── Combat stats / Randomize
   ── Combat bars
── Counter list
▸ Country List 国家列表
▸ Presets 预设
296px 中塞入所有这些内容 → 需要大量滚动
问题总结
核心矛盾
1. 兵牌编辑器的"快速路径"和"详情"混在一起
2. Operation Graphics + Unit Counters 共享一个 Strategic Overlay 面板
3. 兵牌列表没有紧凑的缩略模式
4. 编辑状态（创建/修改兵牌）和浏览状态（查看列表）不分离
结果
• 用户需要频繁滚动
• 编辑一个兵牌时看不到其他兵牌
• 信息密度太高，认知负担重

改进方案

A

弹出式编辑器 (Popover/Floating Panel)

点击地图上的兵牌或列表项时，在兵牌附近弹出一个浮动编辑面板（类似 tooltip 但可交互）。侧边栏只保留紧凑的兵牌列表和快速创建按钮。

侧边栏：紧凑列表 → 弹出面板：完整编辑器

B

两级侧边栏 (Summary → Detail)

侧边栏默认显示紧凑的兵牌缩略列表（一行一个：色条 + 国家 + 类型 + 标签）。点击某个兵牌后，侧边栏平滑展开到详情编辑视图，其他列表项折叠。类似手风琴效果。

默认：紧凑缩略列表 → 展开：完整编辑

C

右侧面板 + 左侧列表 (推荐)

左侧边栏保留紧凑的兵牌列表和快速创建功能。右侧边栏（已有 288px）用于显示选中兵牌的完整编辑器。点击兵牌 → 右面板显示详情。这利用了已有的右侧面板（dev_workspace），将编辑器移过去。

左侧：列表 + 快速操作 → 右侧：详情编辑器

D

模态对话框编辑

点击兵牌后弹出一个居中的模态对话框，包含完整的编辑界面。侧边栏只显示列表和快速操作。编辑完毕后关闭对话框。

侧边栏：仅列表 → 模态框：完整编辑
