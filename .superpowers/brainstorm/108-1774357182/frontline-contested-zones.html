争议/混乱占领区域的表达

当前系统只有"A控制"或"B控制"的二元状态。如何表达中间态？

典型场景

中日战争 1939
日军控制铁路线和主要城市
农村和山区处于游击区/无人区
伪满洲国区域名义控制 vs 实际渗透
→ 需要"渐变"和"混合"的表达
东线 1943-44
大规模推进中的前线不是一条线
而是一个宽数十公里的"混战带"
包围圈、突出部、分散的抵抗
→ 需要"区域化"的前线概念

表达方案

A

条纹 / 斜线填充 (Hatching)

在争议区域的 feature 上叠加斜线纹理，用双色交替条纹表示两方均有存在。类似传统地图中的"争议领土"画法。可以结合现有 controller 数据，标记某些 feature 为 "contested" 状态。

B

前线渐变带 (Gradient Band)

在前线两侧渲染一个可控宽度的渐变色带，从一方颜色过渡到另一方颜色。带越宽表示前线越不稳定/混乱。可以局部调整带宽——稳定段窄，混战段宽。

C

控制程度层级 (Control Level)

扩展 controller 数据模型，每个 feature 除了"谁控制"外，增加"控制程度"字段（0-100%），通过颜色透明度/饱和度来体现。100% = 完全控制（实色），50% = 争议（半透明），0% = 无控制。

D

暂不需要此功能

先专注于前线形状改进，混乱区域表达可以后续再考虑。当前优先级不高。
