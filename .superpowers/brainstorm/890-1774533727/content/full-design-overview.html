Four changes to the strategic overlay system. Review each section below.

1. Workspace Backdrop — Fix Stacking

Before (Bug)
MAP
Workspace Modal
Blur leaks onto modal!
Sidebar stacking context traps z-index
After (Fix)
MAP
Workspace Modal
Crisp, no blur!
Modal moved to document.body on open
Root cause: The modal (#strategicOverlayPanel) lives inside the sidebar DOM. If the sidebar creates a stacking context, the modal's z-index: 46 is trapped, rendering below the backdrop's z-index: 44 at the body level.
Fix: When opening, move the modal element to document.body (escaping any parent stacking context). On close, move it back to the sidebar. This is a standard modal pattern.

New formula:

Key change: Remove the zoom^0.08 exponent that flattened scaling. Instead, let desiredScreenScale track zoom linearly up to the 2× cap. Below zoom 0.5, fade opacity; below 0.3, hide entirely.
