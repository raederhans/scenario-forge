# 导出流程 QA 验收脚本

> 目标：覆盖导出硬性约束、失败分级提示与关键一致性行为。

## 0. 预置环境

1. 启动应用（本地开发模式）。
2. 打开任意包含多图层与文本标注的场景。
3. 打开 **Project → Export** 区块，再进入 `Open workbench`。

## 1. 导出硬性约束

### Case P：真实预览可用

1. 打开 Export workbench。
2. 确认预览区显示真实画面，而不是占位文案。
3. 切换 `Main image / Single layer`。
4. 预期：
   - 两种模式都显示真实预览。
   - `Single layer` 会随预览来源切换而变化。

### Case T：Text stack 可见性真实生效

1. 在 `Text elements` 里分别关闭 `Render-pass labels` 与 `SVG annotations`。
2. 观察预览，再执行一次导出。
3. 预期：
   - 预览会立刻反映文字层变化。
   - 导出结果与当前文字层可见性一致。

### Case A：8K 上限约束

1. 保持导出倍率为 `4×`（或能触发超过 8K 的倍率）。
2. 点击 `Download Snapshot`。
3. 预期：
   - 不触发下载。
   - 出现「参数错误」级别提示。
   - 提示中包含可恢复操作（调整倍率/格式后重试）。

### Case B：并发导出上限约束

1. 连续快速点击 `Download Snapshot` 两次以上。
2. 预期：
   - 第一条导出可执行。
   - 后续请求被拒绝并提示「导出队列已满」。
   - 当前任务完成后，可再次导出。

### Case C：预览分辨率与最终导出分辨率分离

1. 记录当前预览画布尺寸（例如 1920×1080）。
2. 选择导出倍率 `2×` 并导出。
3. 检查导出文件像素尺寸。
4. 预期：
  - 预览尺寸不变。
  - 导出文件尺寸为预览的 2×（例如 3840×2160）。

### Case B1：Bake current pack

1. 点击 `Bake current pack`。
2. 预期：
   - Bake 列表出现更新时间/缓存状态变化。
   - 不报错时，后续 `Bake pack` 导出可直接生成多文件结果。

### Case B2：Bake pack 多文件导出

1. 将 `Target` 切换到 `Bake pack (v1.1)`。
2. 点击下载。
3. 预期：
   - 浏览器会下载多张 baked PNG 与一个 metadata JSON。
   - 成功提示是 bake-pack 语义，而不是普通 snapshot 语义。

## 2. 导出失败分级提示

### Case D：参数错误

1. 使用会触发尺寸超限的导出配置。
2. 预期：
   - 提示标题为参数错误级别。
   - 文案包含「检查分辨率倍率/格式并重试」。

### Case E：内存不足

1. 在高分辨率与高负载场景下导出（可通过多开高内存页面提高触发概率）。
2. 预期：
   - 提示标题为内存不足级别。
   - 文案给出恢复建议：降低分辨率、关闭高占用页面、重试。

### Case F：跨域 SVG 导出失败

1. 在 SVG 叠加层引入跨域资源（不带可用 CORS 头）。
2. 执行导出。
3. 预期：
   - 提示标题为跨域 SVG 失败级别。
   - 文案给出恢复建议：改同源资源/移除跨域资源/临时关闭 SVG 叠加层。
