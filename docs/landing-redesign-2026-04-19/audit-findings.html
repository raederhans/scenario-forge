严重问题 (Critical)

C-1

Hero 产品截图缺乏"主角感" — 直接放图，无悬浮/深度感

当前截图直接铺在 product-frame 内，背景 #0c141f 虽深，但卡片本身没有投影层次，截图与页面背景的关系是"贴在纸上"，不是"漂浮在空间里"。对比 OpenAI 的产品页，截图有大面积暗色背景 + 精确 box-shadow(0 40px 80px) + 轻微 3D 透视角度，产生强烈的产品展示感。

改进方向加深 product-frame 背景到纯 #000 或深蓝，增大 box-shadow (0 60px 120px rgba(0,0,0,0.4))，考虑为截图区域增加顶部光晕 gradient overlay，让截图"发光"而非"平铺"。
严重
C-2

同一张截图在 Hero 和 Works 第一张卡片重复出现

workspace-overview.webp 同时用于 Hero 区产品展示和 Works Section 第一张宽卡片 (work-card--wide)。用户滚动几秒就会再次看到完全相同的图，这在专业产品页面中几乎不会出现。Anthropic、OpenAI 的每一个 section 都有独立的视觉资产。

改进方向Works section 必须使用独立截图资产，或考虑将 Hero 换成更"产品印象"的抽象视觉（渐变、粒子、交互演示），Works 才展示真实截图。
严重
C-3

页面留白严重不足 — 节奏密集，无"呼吸感"

当前 .section { padding: 52px 0 } 与 .section--muted 的上下 margin: 28px，在 1240px 宽屏下各 section 之间几乎无视觉间距。对比 Apple 官网，section 之间通常有 100px–200px 的空白，让每个内容块有独立的视觉重量。当前页面感觉"一直在说话，没有停顿"。

改进方向.section padding 至少提升到 80px–120px 0，关键 section 如 Hero 增加 140px+ 顶部空间，section--muted margin 增加到 48px+。
严重
C-4

零动效 — 页面完全静态，缺乏现代产品感

整页除了按钮 hover translateY(-1px) 外没有任何动效。OpenAI、Anthropic、Apple 的产品页都使用 scroll-triggered 淡入/上移动画（IntersectionObserver + CSS transition），这让用户感受到内容"被揭示"的节奏感，是现代 SaaS/产品页的基本期望。

改进方向为所有 section-heading、卡片添加 data-animate 属性，通过 IntersectionObserver 在进入视口时触发 opacity: 0→1 + translateY(24px→0) 的 400ms ease-out 动画。
严重

重要问题 (Major)

M-1

Eyebrow 标签风格过时 — 橙色大写 + 大字间距

每个 section 都有 .eyebrow { color: #9a5d31; font-weight: 700; letter-spacing: 0.12em; text-transform: uppercase } 的标签。这是 2015–2020 年 SaaS 网站的典型风格。Anthropic 和 OpenAI 几乎不使用橙色 eyebrow，而是用轻量的灰色或配色系统中的强调色，且减少 letter-spacing 至 0.05em 以内。

改进方向eyebrow 改为 color: var(--text-tertiary)，font-weight: 500，letter-spacing: 0.04em，去掉全大写或改为首字母大写。
重要
M-2

Features/Audience/Roadmap 三个 section 视觉上几乎一样

这三个 section 共用相同的卡片样式（白色半透明 + 圆角 + 文字），无图标、无截图、无视觉区分。用户在视觉上无法区分这些内容的重要性差异。Apple 产品页每个 section 都有独特的视觉"结构"。

改进方向Features section 增加 feature icon（SVG 图标，每个 feature group 配一个）。Audience section 改为更大字体的有序列表样式。Roadmap 增加进度条或状态 badge。
重要
M-3

Why/Comparison section 视觉太弱 — 两个白盒子 + 列表

Problem vs. Solution 对比是产品页的核心说服时刻，但当前实现只是两个颜色略有差别的白卡片，里面是 bullet list。OpenAI 的同类对比会用 ❌/✓ icon、颜色对比强烈的卡片来强调差异。

改进方向Problem 侧加 ✗ prefix + 红色色调（#fff5f5），Solution 侧加 ✓ prefix + 蓝色色调（#f0f7ff），增加彩色图标前缀提升说服力。
重要
M-4

Hero facts list 感觉像 README 而非营销文案

三条 hero__facts 列表条目用的是带 border 的白色圆角小盒子，内容是密集的功能说明句。这在视觉上打断了 hero 区的叙事节奏，让人感觉进入了"技术文档"模式。

改进方向将 hero facts 完全移除，详情留给 Features section。Hero 只保留 eyebrow badge + H1 + subtext + 2个CTA。
重要
M-5

Works section 宽卡片文字区过小且内容空洞

work-card--wide 的文字区 padding: 24px 感觉像附注，而非主要说明。label + h3 + p 三层文字塞在窄区域内，H3 的 font-size 被 clamp(1.4rem, 2vw, 2rem) 限制，在宽屏下偏小。

改进方向改用 Bento Grid 布局（1张大卡 + 2张小卡），截图作背景，文字叠加在底部渐变蒙层上。参考 Apple/Linear 产品展示页。
重要
M-6

导航栏信息密度过高 — 品牌 + 4个nav + 语言切换 + 2个按钮

当前 topbar 有 8 个交互元素，Apple 的导航通常只有 品牌 + 核心导航 + 1个CTA，语言切换被收入 footer 或下拉菜单。过多的 topbar 元素分散了用户对主 CTA "Open demo" 的注意力。

改进方向将语言切换移至 footer，topbar 精简为 品牌 + 导航 + GitHub + 1个主CTA（共 7 元素 → 5 元素）。
重要

优化点 (Minor)

P-1

H1 字间距 letter-spacing: -0.06em 在小屏幕上难以阅读

在 3rem (clamp 下限) 时 -0.06em 的字间距会让英文在小屏幕上偏紧。

改进方向用 @media 在小屏幕下将 letter-spacing 放宽到 -0.03em。
优化
P-2

CTA Panel 左右布局在移动端 stacked 后按钮组很突兀

cta-panel__actions 在折叠后上下排列按钮，但 CTA 文字区的 h2 相对较大，整体比例失调。

改进方向改为居中单列布局：eyebrow + h2 + p 居中，按钮组居中 flex。
优化
P-3

button--primary 颜色 #18222f 无品牌个性

当前主 CTA 按钮是几乎纯黑色，在奶油色背景下稍显生硬，且与品牌色 #234d83 无关联。

改进方向将 button--primary 改为 var(--accent) (#234d83 深蓝) + 白色文字，与页面品牌色统一。
优化
P-4

footer 结构不清晰 — 两列无分隔，数据归因段落冗长

footer__meta 中的数据来源文字是一段密集的散文，难以扫读。

改进方向添加顶部分隔线，数据来源改为轻量小字（0.78rem, color:#aaa），语言切换移入 footer。
优化
P-5

Works section 缺少宏观标题的视觉重量

section-heading max-width: 760px 且居左，标题字号在 clamp 最小值时缺乏足够的视觉引导作用。

改进方向section-heading 居中对齐，h2 字号提升至 clamp(2.6rem, 5vw, 4.5rem)。
优化

当前 vs 目标 — Hero 区对比示意

当前状态

截图平铺；Hero facts 列表占据大量空间；截图投影不足；chips 覆盖截图。

目标方向（参考 OpenAI/Apple）

深色首屏；居中文字；截图从底部升起有光晕；无 facts 列表。
