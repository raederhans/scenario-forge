# 铁路图层方法论归档

日期：2026-03-28

## 1. 目标

这份文档用于归档 `铁路图层` 的通用方法论，作为后续不同国家试点时的统一口径。

当前方法论只面向：

- 地图中的 `半业务线图层`
- 当前版本铁路网络
- `铁路线 + 主要车站`
- 可延迟加载的 context layer

当前方法论明确不面向：

- 连续历史时间轴
- 班次、运行图、routing
- 全量车站系统
- 客货流量分析
- 铁路编辑器

## 2. 产品定位

铁路图层首版不是“把所有轨道对象都画出来”，而是做一个可读、可筛选、可保存的铁路网络图层。

首版必须满足：

- 可单独开关
- 可按状态筛选
- 可按等级筛选
- 可点击查看线路属性
- 可显示主要车站
- 可点击查看车站属性
- 可随项目导出和恢复

## 3. 核心方法论

### 3.1 先把线路和主要车站分开定义

铁路首版固定研究两类对象：

- 铁路线
- 主要车站

不把以下对象混进首版：

- 全量站点
- 站场细部
- 线路运行关系
- 客货运组织

### 3.2 首版默认只做当前网络与生命周期状态

铁路的产品价值首版来自：

- 当前网络骨架
- 生命周期状态
- 可解释的等级映射

默认状态固定为：

- `active`
- `disused`
- `abandoned`
- `construction`

首版不做年份快照，不做连续时间轴。

### 3.3 等级统一为产品显示等级，不直接照搬各国法律分类

跨国铁路图层首先要稳定显示，不是先追求每国制度完全等价。

默认等级固定为：

- `high_speed`
- `trunk`
- `branch`
- `service`

它们是 `产品显示等级`，不是各国法规术语原样照抄。

### 3.4 铁路允许三种主源策略，但每个国家只能明确选一种

后续国家研究只允许三种结论：

1. `官方主源 + OSM 补缺`
2. `OSM 主几何 + 官方属性加固`
3. `官方全国主源缺口成立`

不允许把这三种策略混成模糊的“多源综合”。

### 3.5 主要车站优先，不先做全站点

主要车站的目标是帮助全国尺度下建立可读节点层。

默认优先纳入：

- 首都核心站
- 主要大城市核心站
- 与 `high_speed` 线路直接相关的重要站
- 行政或区域级核心换乘站

默认不先追求全站覆盖。

## 4. 数据策略

### 4.1 官方主源的作用

如果一个国家存在成熟官方主源，优先让它承担：

- 现役线路主几何
- 官方线路命名
- 官方站点命名
- 官方结构口径

### 4.2 OSM 的作用

OSM 的典型价值主要在：

- `disused`
- `abandoned`
- `construction`
- 官方明显缺失的边缘要素

如果一个国家最终走 `官方主源 + OSM 补缺`，就不允许把 OSM 现役网络全量盖到官方主源上。

### 4.3 走 `OSM 主几何 + 官方属性加固` 的前提

只有当一个国家没有足够成熟、公开、全国统一的铁路官方几何主源时，才允许采用：

- OSM 或 Geofabrik 作为当前主几何
- 官方专题源做线路身份、编号、运营主体或分类加固

### 4.4 明确缺口优先于勉强补齐

如果一个国家没有强官方主源，也没有足够稳的官方加固链，就应直接写成：

- `官方全国主源缺口成立`

而不是用启发式补丁把研究结论伪装得很完整。

## 5. 推荐架构

### 5.1 接入方式

首版统一推荐：

- `独立 deferred context layer pack`
- `railways` 线图层
- `rail_stations` 点图层

不建议：

- 塞进主拓扑
- 和行政主数据混编
- 把铁路、车站、客流、班次揉成一个大层

### 5.2 前后端职责划分

后端负责：

- 读源数据
- 统一 CRS
- 字段标准化
- 生命周期状态映射
- 等级映射
- 严格去重
- 输出稳定 pack

前端负责：

- 懒加载
- 按状态与等级显示
- 车站显隐控制
- 样式
- 点击查询
- 项目状态持久化

### 5.3 推荐的内部统一 schema

线路字段建议固定为：

| 字段 | 含义 |
|---|---|
| `id` | 稳定主键 |
| `name` | 线路名 |
| `source` | 数据来源 |
| `status` | `active/disused/abandoned/construction` |
| `class` | `high_speed/trunk/branch/service` |
| `operator` | 运营主体，可选 |
| `geometry` | 线几何 |

主要车站字段建议固定为：

| 字段 | 含义 |
|---|---|
| `id` | 稳定主键 |
| `name` | 站名 |
| `source` | 数据来源 |
| `city_key` | 所属城市键 |
| `importance` | 站点重要度 |
| `geometry` | 点几何 |

## 6. 跨国试点选择原则

一个国家适不适合做铁路试点，优先看五件事：

1. 有没有全国统一、公开、可解释的铁路主网络源
2. 是否能稳定获得主要车站信息
3. 生命周期状态能不能被明确映射
4. 许可边界能不能直接写清
5. 字段能不能映射到统一 schema

如果一个国家满足不了前 3 条，就不要勉强宣称“已经具备强官方主源”。

## 7. 首版验收标准

铁路图层首版通过的最低标准：

1. 图层能独立懒加载
2. 线路状态和等级规则可预测
3. 主要车站不会退化成全国站点名录
4. 点击属性能稳定显示
5. 项目导入导出后状态可恢复
6. 不影响现有 context layer 的加载性能

## 8. 一句话结论

铁路图层的正确起点不是“先把所有铁路事实收齐”，而是：

`先把当前铁路主网络和主要车站做成一个可解释、可筛选、可保存的图层，再根据各国现实在“官方主源 + OSM 补缺”“OSM 主几何 + 官方加固”与“缺口成立”三种策略里明确选一条。`
