# 交通工作台高密图层表达重构执行稿

日期：2026-04-02

## 1. 目标

本稿用于执行交通工作台中高密图层的表达重构，范围覆盖：

- 港口
- 矿产资源
- 能源设施
- 工业用地
- 物流中心

本次不再把问题只停留在前端样式层，而是同时覆盖：

- workbench 右侧 tab 交互
- family preset schema
- 聚合 / 密度 / 审查三模式
- 三级地理标签引擎
- 港口口径与矿产分类等构建链契约

## 2. 核心结论

- 主路线采用“方案 1”：审查模式、聚合模式、密度模式共存。
- 每个图层单独拥有自己的 preset，但 preset 本质是动态阈值策略，不是固定数量策略。
- 新增控制全部进入右侧 tab，不再向左侧 lens 塞参数。
- 默认仅暴露预设和少量关键旋钮；高级菜单默认收起。
- 标签必须使用真实地理语义，禁止出现 cluster 编号、cell 编号这类机械命名。
- 标签拥有独立布局引擎，必要时允许标签降级和标签聚合。
- 参数必须进入项目保存链路，支持后续继续调参。

## 3. 右侧 Tab 结构

右侧 inspector 扩展为以下结构，统一服务当前 family：

1. `Inspect`
   - 显示当前选中对象、聚合单元或密度核心的详情。
2. `Display`
   - 切换 `inspect / aggregate / density`。
3. `Aggregation`
   - 切换算法、preset、少量关键阈值入口。
4. `Labels`
   - 控制标签层级、预算、显示开关，详细阈值进入高级菜单。
5. `Coverage`
   - 仅在有覆盖口径需求的 family 出现，例如港口。
6. `Data`
   - 展示 manifest、build audit、source track、当前模式统计，只读。

左侧 lens 继续只负责上下文预览和空间观察，不承担参数控制。

## 4. Family preset schema

每个 family 的配置统一扩展为：

- `mode`
- `preset`
- `aggregation`
- `labels`
- `coverage`
- `filters`

其中：

- `mode`：当前显示模式。
- `preset`：当前预设名，例如 `review_first / balanced / pattern_first / extreme_density`。
- `aggregation`：算法、单元尺寸、切模式阈值、极密触发阈值、类别集中度阈值。
- `labels`：标签层级、预算、分离强度、标签聚合阈值、命名规则。
- `coverage`：港口等图层的覆盖口径。
- `filters`：原有筛选字段，继续保留。

保存策略采用：

- 内置默认 preset
- 项目 override

项目只记录用户修改过的字段，不冗余保存整份默认配置。

## 5. 动态阈值原则

所有模式切换与标签控制均采用动态阈值，不允许写死“前 N 个”：

- 视口对象拥挤度
- 屏幕重叠率
- 标签冲突率
- 聚合单元内部类别集中度
- 局部极密强度
- 当前缩放级别
- 当前筛选状态

切换逻辑固定为：

- 审查模式拥挤时切聚合模式
- 聚合模式继续拥挤时切密度模式
- 用户细筛或放大后自动回退到更可审查模式

## 6. 各图层默认算法

### 6.1 港口

- 默认：`inspect`
- 口径：`core / expanded / full_official`
- 低缩放：仅在扩展或全量口径下启用轻量 `cluster`
- 不启用连续热力图

### 6.2 矿产资源

- 默认：`aggregate`
- 主算法：`hex grid`
- 备选：`square grid`
- 默认流程：先按类别筛选，再聚合
- 保留 `inspect` 回看单点

### 6.3 能源设施

- 默认：`inspect`
- 必做：subtype 着色
- 低缩放可选：`cluster`
- 首批不做连续密度层

### 6.4 工业用地

- 默认：`aggregate`
- 主算法：`square grid`
- 备选：`density surface`
- `internal / open` 双轨独立计算，禁止混算

### 6.5 物流中心

- 默认：`aggregate`
- 低缩放：`cluster`
- 中缩放：`grid`
- 高缩放：回 `inspect`

## 7. 三级地理标签引擎

标签分三层：

1. 一级：大区级
   - 例如 `关东 / 关西 / 东北 / 北九州`
2. 二级：区域锚点级
   - 例如 `东京湾沿岸 / 仙台周边 / 濑户内沿岸`
3. 三级：锚点 + 主类别
   - 例如 `东京湾沿岸·物流`、`东北中部·金属矿产`

标签生成流程：

1. 先匹配真实地理锚点。
2. 再计算单元内部主类别或混合类别。
3. 再按标签预算、缩放、冲突率决定输出到哪一级。

标签布局引擎流程：

1. 常规避让
2. 标签降级
3. 标签聚合
4. 极密区域使用上位地理标签

密度模式下默认只输出少量代表性标签，不追求全覆盖。

## 8. 数据构建链

### 8.1 港口

- 从当前 `317 -> 12` 的主动裁剪升级为三档 coverage 输出。
- manifest 和 build audit 明确写出三档数量和裁剪规则。
- 前端只消费 coverage 结果，不在运行时猜口径。

### 8.2 矿产资源

- 保留原始 `resource_type`
- 新增稳定 `normalized_resource_group`
- 前端默认按 group 做筛选和聚合
- inspector 继续可见原始 type

### 8.3 能源设施

- 继续使用现有 subtype catalog
- reference-only subtype 保持显式状态，不混入 live 结果

### 8.4 工业用地

- 保持 `internal / open` 双轨 manifest 契约
- 聚合结果按当前轨道独立生成

### 8.5 物流中心

- 复用现有 build audit 的地域统计
- 为标签锚点和聚合阈值补充辅助字段

## 9. 实施顺序

### 第一阶段：工作台表达升级

- 右侧 tab 架构
- family preset schema
- 模式切换与算法接入
- 标签引擎第一版
- 项目持久化接入

### 第二阶段：构建链治理

- 港口 coverage 三档
- 矿产 normalized group
- build audit / manifest 增补聚合友好字段
- 地理锚点数据支持

## 10. 验收标准

1. 右侧 tab 完整接入，左侧无新增参数堆积。
2. 参数默认少暴露，高级菜单默认收起。
3. 所有模式切换基于动态阈值，不存在固定数量逻辑。
4. 标签命名只出现真实地理语义，不出现算法编号。
5. 港口 coverage 三档、矿产分组、能源 subtype 着色、工业双轨隔离都能在 inspector 中被验证。
6. project save/load 能恢复图层模式、preset、标签和 coverage 设置。

## 11. 一句话结论

这次改造的核心不是“给现有点层再补一点样式”，而是把高密图层正式升级为“可审查、可聚合、可读图、可持久化”的表达系统。
