# 公路图层方法论归档
日期：2026-03-28

## 1. 目标

这份文档用于归档 `公路/高速公路图层` 的通用方法论，作为后续不同国家试点时的统一口径。

当前方法论只面向：

- 地图中的 `半业务线图层`
- 当前版本路网
- 可延迟加载的 context layer

当前方法论明确不面向：

- 历史回溯
- routing
- 交通流量分析
- 路网编辑器
- 全量道路细节可视化

## 2. 产品定位

公路图层首版不是“把所有路都画上去”，而是做一个可读、可筛选、可保存的主干路网图层。

首版必须满足：

- 可单独开关
- 可按等级筛选
- 可按缩放分层显示
- 可点击查看基础属性
- 可随项目导出和恢复

## 3. 核心方法论

### 3.1 先定义产品等级，不先追数据全量

公路图层先定义 `产品需要显示什么等级的路`，再去选择数据源。

默认采用 `功能等级路网`，而不是直接采用各国行政分类。

原因很简单：

- 跨国更容易统一
- 更适合地图可视化
- 更适合缩放分层
- 不会被各国法定等级体系绑死

### 3.2 首版默认只做当前快照

公路部分不做历史回溯，不做时间轴，不做多年份快照。

原因：

- 公路网络历史整理成本高
- 历史口径比铁路更不稳定
- 当前产品价值优先来自“现在的主干路网”

### 3.3 主干路网优先，低价值支路默认排除

公路图层首版必须优先控制密度。

默认优先级：

- `motorway`
- `trunk`
- `primary`

默认不进入首版主图层：

- `*_link`
- `service`
- `residential`
- `track`
- 其他明显只会制造噪音的支路

### 3.4 公路数据策略默认是“几何主源”和“属性加固”分离

铁路试点可以较自然地走“官方主源 + OSM 补缺”。

公路通常不能机械照搬。原因是很多国家的官方开放道路数据会出现以下情况：

- 全国网不完整
- 只有专题道路
- 几何太旧
- 可商用边界不清晰
- 真正高质量全国路网是收费产品

因此，公路图层的默认策略是：

- 用一个统一、当前、全国覆盖好的源做 `几何主源`
- 用官方专题源做 `属性加固`

在开放数据世界里，这个几何主源通常会落到 `OSM / Geofabrik`。

### 3.5 公路首版必须把降噪放进一阶段

公路和铁路最大的工程差异，不在于加载方式，而在于公路的密度远高于铁路。

所以首版就必须做：

- 等级过滤
- 缩放分层
- link/service 排除
- 最小视觉长度阈值
- 安全的连续线合并

首版明确不做：

- 激进启发式路网折叠
- 模糊规则合并
- 用“后处理补丁”去掩盖源数据问题

## 4. 推荐架构

### 4.1 接入方式

首版推荐统一走：

- `独立 deferred context layer pack`
- `独立 roads 线图层`
- `独立 road labels 图层`

不建议：

- 塞进主拓扑
- 进入 scenario chunk
- 与行政边界主数据混编

### 4.2 前后端职责划分

后端负责：

- 读源数据
- 统一 CRS
- 字段标准化
- 严格过滤等级
- 严格去重
- 安全合并连续线段
- 输出稳定 pack

前端负责：

- 懒加载
- 按缩放分层显示
- 线样式
- 编号样式
- 点击查询
- 项目状态持久化

### 4.3 推荐的内部统一 schema

首版道路线图层建议固定字段为：

| 字段 | 含义 |
|---|---|
| `id` | 稳定主键 |
| `name` | 道路名称 |
| `ref` | 道路编号 |
| `road_class` | `motorway/trunk/primary` |
| `source` | 数据来源 |
| `official_ref` | 官方编号或校正编号，可选 |
| `geometry` | 线几何 |

如果有编号图层，建议单独做 label 候选，不要直接把所有文字渲染逻辑绑在线图层上。

## 5. 跨国试点选择原则

一个国家适不适合做公路试点，优先看五件事：

1. 有没有当前、全国、公开可用的主干路网几何
2. 有没有可用于校正的官方专题道路数据
3. 数据许可能不能直接进入产品
4. 高密城市地区能不能稳住可读性
5. 数据字段能不能稳定映射到统一 schema

如果一个国家满足不了这五条里的前三条，就不要强行作为首批试点。

## 6. 首版验收标准

公路图层首版通过的最低标准：

1. 图层能独立懒加载
2. 低缩放下不会糊成一片
3. `motorway/trunk/primary` 的显隐规则可预测
4. 编号不会大面积重叠失控
5. 项目导入导出后状态可恢复
6. 不影响现有 context layer 的加载与性能

## 7. 一句话结论

公路图层的正确起点不是“收更多路”，而是：

`先用功能等级把当前主干路网做成一个可读的线图层，再在明确可解释的前提下逐步加官方属性和更细道路。`
