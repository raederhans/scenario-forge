# 美国产业矿产研究草稿

日期：2026-03-28

## 一句话结论

如果美国这条线的目标是 `矿床/资源分布` 而不是“当前在产矿山”，那么最稳的做法不是找单一主表，而是用 USGS 官方组合源落一版全国骨架：`MRDS + Alaska ARDF + USMIN 增强`。

## 研究边界

- 研究对象是 `矿床 / 矿化点 / prospects / occurrences / resource distribution`
- 首版几何以 `点` 为主
- 不要求反映当前是否在产、是否有采选设施、是否有现势储量
- 不做矿权范围、矿区面、尾矿库、运输通道

## 主源 / 后备源 / 排除项

### 主源

- `MRDS` 作为全美广覆盖点位骨架
- `ARDF` 作为阿拉斯加替换源，而不是阿拉斯加后备源
- `USMIN` 作为重点矿床、关键矿种、重要矿区的官方增强层

这条线的正确结论不是“美国有一张像日本 GSJ 那样干净的全国主表”，而是“美国有一套足够强的官方组合主源”。

### 后备源

- USGS 其他专题 deposit / mineral-system 数据库，仅在需要加细分矿种时再引入
- 州级地调或矿业监管公开数据，只用于局部补点，不反过来重写全国口径
- Tier B 本轮没有找到比 USGS Tier A 更适合的全国统一候选

### 排除项

- 现役矿山新闻、公司矿山清单、矿业协会名录
- 产量、储量、资本市场披露
- 矿权范围、采矿权边界、矿山设施分布
- 用企业地址或矿权中心点反推“矿床分布”

## 与现有仓库架构的承接判断

这条线仍然最适合走点图层。

- `MRDS` 和 `ARDF` 直接就是点记录，能直接接入现有点图层链路。
- `USMIN` 中的面、矿区、矿带记录，不应把仓库改造成面层主逻辑；更稳的做法是在入库阶段生成 `representative_point`，并保留原始几何来源字段。
- 首版字段可维持最小集合：`id`、`name`、`resource_type`、`resource_class`、`source`、`geometry`。

## 与日本最明显的不同

日本这条线更接近“一张 GSJ 全国专题表就能跑通”；美国这条线更像“USGS 官方体系内的三段式拼接”。

- 日本主源更整洁。
- 美国权威性并不弱，甚至总量更强，但它是按历史库、阿拉斯加专库、重点矿床库拆开的。
- 所以美国首版更依赖数据整合规则，而不是单纯找一张完美主表。

## 风险与下一步建议

1. 最大风险不是缺数据，而是把这条线误写成“美国现役矿山图层”。
2. `MRDS` 很适合当骨架，但不能拿来承诺现势状态；`USMIN` 很权威，但不能拿来承诺全国全矿种全覆盖。
3. 建议的最短路径是：
   1. 以 `MRDS` 拉起全美点位骨架。
   2. 用 `ARDF` 替换阿拉斯加部分。
   3. 用 `USMIN` 给关键矿种和重点矿床打增强标签。
   4. 在产品文案中把名称固定成 `矿床/资源分布`，不碰经营状态。
