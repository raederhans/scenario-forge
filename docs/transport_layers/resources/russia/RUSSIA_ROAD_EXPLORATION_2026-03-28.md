# 俄罗斯公路专题研究归档

日期：2026-03-28

## 1. 一句话结论

俄罗斯公路这条线当前最稳的判断是 `OSM 主几何 + 官方属性加固`：交通部和 Росавтодор 对 федеральные трассы 与 `опорная сеть автомобильных дорог` 有官方定义和持续更新的建设口径，但没有查到一份公开、全国、可直接产品化的主干道路 GIS 主源，因此几何现实上仍要依赖 OSM。

## 2. 研究边界

- 只研究 `最新快照`
- 只研究 `motorway / trunk / primary`
- 不研究历史回溯
- 不研究 routing、收费、匝道体系和复杂节点
- 不研究 `secondary` 及以下道路

## 4. 主源 / 后备源 / 排除项

### 4.1 主源

- 几何主源：`OSM / Geofabrik Russia`

### 4.2 后备源

- Минтранс / Росавтодор 对 `опорная сеть` 与 федеральные трассы 的官方口径
- 用于：
  - 判断哪些道路应优先保留
  - 对高等级道路进行官方语义加固

### 4.3 排除项

- 历史道路变迁
- 城市快速路和地方路网的全量并表
- 匝道、收费站、服务区等复杂节点系统

## 5. 欧洲俄罗斯与非欧洲部分的差异

- `欧洲俄罗斯`
  - федеральные трассы 和主城市群之间的高等级道路更集中
  - 官方 `опорная сеть` 口径更容易转译成首版主干道路层
- `非欧洲俄罗斯`
  - 西伯利亚和远东的长距离走廊仍然重要，但路网稀疏，分类和现势性更依赖 OSM
  - 因此东部比欧洲俄罗斯更明显地依赖 Tier C 几何主源

## 6. 与现有仓库架构的承接判断

- 继续走 `roads` + `road_labels` 两个独立 pack
- 首版的安全定义应是“俄罗斯主干公路上下文层”
- 构建时优先保留：
  - 官方明确属于 `опорная сеть` 或 федеральные трассы 的主走廊
  - OSM 中对应 `motorway / trunk / primary` 的主干网

## 7. 与日本最明显的不同

- 日本公路更像 `OSM 主几何 + 官方高速加固`；俄罗斯公路更像 `OSM 主几何 + 官方国家骨架口径加固`。
- 日本的官方高速专题更贴近业务对象；俄罗斯官方信息更偏政策和建设口径，而不是开放 GIS 成品。
- 日本全国密度更高、图层更容易均匀；俄罗斯在东部天然需要更强的放宽。

## 8. 风险与下一步建议

### 8.1 风险

1. 最大风险是把 `опорная сеть` 的政策定义误写成已经存在公开 GIS 主层。
2. 第二个风险是把东部长走廊的低密路网和欧洲俄罗斯同等对待，导致图层目标过满。
3. 第三个风险是为了追求官方纯净而放弃最现实的 OSM 几何方案。

### 8.2 下一步建议

1. 公路首版直接接受 `OSM 主几何 + 官方属性加固`。
2. 先把欧洲俄罗斯的主干网络跑顺，再把西伯利亚和远东当成长走廊扩展区。
3. 不要追求一层包打尽“联邦 + региональные + 城市级”全体系。
