# 俄罗斯铁路专题研究归档

日期：2026-03-28

## 1. 一句话结论

俄罗斯铁路这条线当前最稳的结论是 `官方全国主源缺口成立`：РЖД 和交通主管体系能给出全国铁路网络的官方描述、规模和重点走廊，但公开、可直接产品化的全国铁路线几何和主要车站点位主源并不整齐，因此真要落地图层时只能接受 `OSM 主几何 + 官方背景加固` 的现实路径。

## 2. 研究边界

- 研究对象固定为 `铁路线 + 主要车站`
- 不研究时间轴
- 不研究班次、routing、运行图
- 不研究全量车站设施系统
- 不研究地铁、轻轨、工业专用线的完整专题化并表

## 4. 主源 / 后备源 / 排除项

### 4.1 主源

- 研究结论层面：`官方全国主源缺口成立`
- 真正落地图层时的现实几何入口：`OSM / Geofabrik Russia`

### 4.2 后备源

- РЖД 官方公开报告和网络说明，用于核对：
  - 网络规模
  - 电气化比例
  - 重点干线和区域覆盖

### 4.3 排除项

- 班次、运行图、调度数据
- 全量站场设施
- 运营级里程点、桥隧、信号与 capacity 数据
- 把公开企业报告误写成现成 GIS 主源

## 5. 欧洲俄罗斯与非欧洲部分的差异

- `欧洲俄罗斯`
  - 主网密度更高，主干线和主要城市节点更容易靠 OSM + 官方网络说明稳定收口
  - 主要车站的筛选也更容易围绕莫斯科、圣彼得堡、下诺夫哥罗德、喀山、罗斯托夫等大节点开展
- `非欧洲俄罗斯`
  - 西伯利亚和远东的主干线仍然重要，但站点稀疏、支线更弱，实际落地时更依赖 Tier C
  - 因为东部缺口，非欧洲部分必须接受更强的 `OSM / Geofabrik` 降级使用

## 6. 与现有仓库架构的承接判断

- 继续走 `railways` + `rail_stations` 两个 deferred context pack
- `railways` 首版只承诺全国主干铁路骨架，不承诺运营级精度
- `rail_stations` 首版只保留主要公共客运站，不追求全站点
- 如果后续真做俄罗斯铁路，建议在构建阶段显式保留 `source_region = european / non_european`

## 7. 与日本最明显的不同

- 日本铁路更接近“官方专题包先在”；俄罗斯更接近“官方网络信息存在，但 GIS 主源缺口明显”。
- 日本可以更自然地走 `官方主源 + OSM 补缺`；俄罗斯当前更诚实的写法是 `官方全国主源缺口成立`。
- 日本的“主要车站”更容易由官方专题层直接筛出；俄罗斯这一步更依赖公开协作源和业务常识筛选。

## 8. 风险与下一步建议

### 8.1 风险

1. 最大风险是把 РЖД 的网络报告误当成可直接落盘的线层主源。
2. 第二个风险是把欧洲俄罗斯的高密主网经验硬套到西伯利亚和远东。
3. 第三个风险是为了追求“全俄官方统一”而拖慢产出。

### 8.2 下一步建议

1. 先接受铁路这条线的官方主源缺口，不再继续追一个不存在的开放全国 GIS 主源。
2. 如果要做首版，直接走 `OSM / Geofabrik` 作为几何底座，官方只负责网络背景和主干线语义。
3. 首版重点先放在欧洲俄罗斯主网，东部保留主干线但弱化精细承诺。
