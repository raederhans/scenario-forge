# 俄罗斯工业矿产、能源设施、工业区研究总览

日期：2026-03-28

## 1. 总体结论

俄罗斯这三条线里，当前最适合先试点的是 `mineral_resources`：全国官方地质与矿产图系最强，虽然提取成本高，但主题最对。`industrial_zones` 可以做，但更像 `官方/准官方园区名录 + 几何补充` 的两层结构；`energy_facilities` 则最容易卡在“官方统计强、官方点位主源弱”，首版必须接受 `power-plants-first` 或 `官方全国主源缺口成立`。

## 2. 三条线总体成熟度判断

| 线别 | 成熟度判断 | 现在能不能直接落盘 | 核心原因 |
|---|---|---|---|
| 工业矿产 | 中高 | 可以，但更像图系与地图服务主源 | Rosnedra 与 VSEGEI 的全国矿产/地质图系最对题，欧洲俄罗斯最稳，非欧洲俄罗斯仍有官方图系覆盖，但更偏中小比例尺和服务式获取 |
| 工业区 | 中 | 可以，但必须写清“名录层 + 几何补充” | 俄罗斯有官方 GISP 技术园/园区体系和强 Tier B 园区目录，但全国统一、开放、直接可用的工业园区 polygon 主源不强 |
| 能源设施 | 中低 | 可以，但要明确收窄边界 | 官方统计、规划和系统资料能支撑类别与规模判断，但全国统一、公开、可直接下载的点状能源设施主源偏弱，东部更容易退到 Tier C |

## 3. 哪条线最适合先试点

最适合先试点的是 `工业矿产`。

原因很直接：

1. 这条线本来就按 `矿床/资源分布` 研究，不要求现役矿山企业清单。
2. 俄罗斯在国家级地质和矿产图系上的传统最强，官方骨架比能源和工业区更稳。
3. 欧洲俄罗斯的数据组织更成熟，远东和西伯利亚即便放宽，也仍能在官方或准官方图系里找到可解释骨架。
4. 这条线的弱点主要是“提取麻烦”，不是“没有主源”。

## 4. 和日本最明显的差异

### 4.1 总体差异

日本更像“全国专题包相对整齐”；俄罗斯更像“官方国家级图系和服务很强，但直接可下载、字段整齐的成品表更弱”。

### 4.2 工业矿产

日本更接近全国资源点或专题产品包；俄罗斯更像全国地质图、矿产分布图和矿权/矿区电子地图体系，语义很强，但抽成统一点表的摩擦更大。

### 4.3 能源设施

日本的能源设施至少能先收敛到全国发电设施锚点；俄罗斯这条线更容易出现“官方统计与规划很强，但点位主源不足”的问题，尤其非欧洲俄罗斯更明显。

### 4.4 工业区

日本工业区更接近真实工业用地专题层思路；俄罗斯更接近“产业园区/技术园区/特区名录 + 几何补充”，园区语义更强，但统一空间几何更弱。

## 5. 哪些地方必须降级到非官方但可信公开源

### 5.1 明确需要降级的地方

- `energy_facilities`：如果首版要求全国点位、跨发电/炼厂/LNG 多子类统一建层，现实上需要降级到 Tier C 的 OSM/OpenInfraMap 或 Global Energy Monitor 这类公开数据库补点位。
- `industrial_zones`：如果首版要求全国园区边界 polygon，而不是命名园区目录，现实上要大量降级到 Tier C 的地区招商目录、园区官网和 OSM 边界补充。

### 5.2 当前不必急着降级的地方

- `mineral_resources` 不必一开始就退到协作源，因为 Rosnedra/VSEGEI 的官方骨架已经足够支撑“资源分布”这个口径。

## 6. 可直接承接到仓库架构的判断

### 6.1 工业矿产

最稳的接法仍是 `resource_areas` 优先，不建议一开始强行纯点化。

### 6.2 能源设施

继续适合点图层，但要把首版边界写窄：

- 最稳的是 `power_plants-first`
- 其他能源设施子类按子类单独补，不要强行压成一张“全能源点层”

### 6.3 工业区

最稳的不是先找全国 polygon 主源，而是：

- `命名园区名录层`
- `边界/位置几何补充层`

两层不要混成看似统一的一张表。

## 7. 风险判断

1. 俄罗斯很多“能公开访问”的国家级资源，本质上是地图服务、图集或名录，不等于可直接落盘的标准主表。
2. 非欧洲俄罗斯不应拖慢整体进度；需要显式接受“欧洲俄罗斯优先，东部放宽”。
3. 工业区最容易误把政策支持对象、园区名录和真实工业用地当成同一件事。
4. 能源设施最容易误把官方统计和运行资料当成可直接产品化的点位主源。

## 8. 下一步建议

1. 先把俄罗斯 `mineral_resources` 跑成正式国家样板。
2. 并行把 `industrial_zones` 收敛成“园区名录主层 + 几何补充”的明确定义。
3. `energy_facilities` 首版不要追求大全层，优先写清 `power-plants-first` 和官方点源缺口。
