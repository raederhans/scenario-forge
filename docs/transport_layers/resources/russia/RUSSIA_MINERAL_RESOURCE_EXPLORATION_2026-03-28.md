# 俄罗斯工业矿产研究归档

日期：2026-03-28

## 1. 一句话结论

如果俄罗斯这条线的目标固定为 `矿床/资源分布` 而不是“现役矿山企业清单”，那么当前最稳的结论是：`官方全国主轴存在，但更像地图服务与图系，不像可直接下载的干净点表`。欧洲俄罗斯最稳，西伯利亚和远东仍有官方/准官方矿产图系支撑，因此不必因为东部缺口而放弃整体主判断。

## 2. 研究边界

- 研究对象是 `矿床 / 资源分布 / 资源潜力`
- 不强求反映当前是否在产
- 不把企业经营主体、产量、储量动态混进首版主层
- 允许首版保留 `resource_areas` 或图系派生点，而不是强行要求原生全国点表

## 4. 主源 / 后备源 / 排除项

### 4.1 主源

- `Rosnedra` 的矿产与矿区电子地图体系
- `VSEGEI` 的国家级地质与矿产图系服务

### 4.2 欧洲俄罗斯与非欧洲俄罗斯

- 欧洲俄罗斯：矿区、矿床与许可区块更容易在 Rosnedra 国家门户里形成稳定主判断。
- 非欧洲俄罗斯：仍有 VSEGEI 的北亚/东部专题图系支撑，不必直接退到 Tier C；但更偏图集、图层和中小比例尺，不如欧洲俄罗斯顺手。
- 只有当目标变成“清洗好的全国点表”时，西伯利亚和远东更容易被迫降到 Tier C 做补点。

### 4.3 后备源

- `VSEGEI` 北亚矿产图集：补强亚洲俄罗斯
- `OSM / Geofabrik Russia`：只在点位抽取、名称补缺或几何缺口时降级使用

### 4.4 排除项

- 现役矿山企业目录
- 只给企业/许可证文本不给几何的名录
- 把矿权区块直接等同于实际在产矿山

## 5. 与现有仓库架构的承接判断

俄罗斯矿产线最稳的承接方式仍是：

- `mineral_resources` 独立资源包
- 主表达优先保留 `resource_areas`
- 若产品端必须点化，再从图层或资源区派生 `representative_points`

不要因为想要快，直接把地图服务截图式结果硬写成点表主源。

## 6. 与日本最明显的不同

- 日本更接近全国专题产品包。
- 俄罗斯更接近“官方国家级矿产图系 + 电子地图入口”。
- 日本的优势是成品感更强；俄罗斯的优势是资源骨架更厚，但工程提取摩擦更大。

## 7. 风险与下一步建议

### 7.1 风险

1. 最大风险是把“官方地图服务存在”误写成“全国标准下载主表存在”。
2. 第二个风险是把东部地区的官方图集型资料误解成现成的统一点源。
3. 第三个风险是把矿产资源分布误写成现势开发活动。

### 7.2 下一步建议

1. 首版先明确写成 `资源分布层`，不要写成矿山企业层。
2. 先以欧洲俄罗斯作为主验证区，再用 VSEGEI 北亚专题确认远东和西伯利亚并非硬缺口。
3. 如果后续实现侧需要全国统一点表，再单独增加一个“图系转点”的构建步骤。
