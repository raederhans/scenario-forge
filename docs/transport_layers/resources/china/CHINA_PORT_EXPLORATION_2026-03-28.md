# 中国港口专题研究归档

日期：2026-03-28

## 1. 一句话结论

中国港口这条线可以先收成 `主要商港/关键港口节点层`，但最稳的结论不是“单一官方点位主层已经完美可用”，而是 `官方规划/统计主锚点 + 点位几何补充`：中国大陆和台湾两边都能找到强官方港口规划、吞吐量和港群资料，但统一的主要港口点层仍需几何补充与人工筛选。

## 2. 研究边界

- 只研究 `设施本体`
- 只做 `点图层优先`
- 只优先收 `主要商港/关键港口节点`
- 不研究航路
- 不研究港域界和港区面
- 不研究渔港专题

## 4. 主源 / 后备源 / 排除项

### 4.1 主源

- 大陆主要港口范围与分类：`水运“十四五”发展规划` + `主要港口总体规划陆续批复`
- 大陆重要度：交通运输部港口吞吐量统计
- 台湾主要港口范围：`國內商港未來發展及建設計畫`
- 台湾重要度：`iMarine` 航港资料

### 4.2 后备源

- `OSM / Geofabrik`：补主要港口点位几何
- 港务公司或港务局公开页面：补个别港口属性

### 4.3 排除项

- 航路
- 港区边界
- 码头级全量设施系统
- 渔港专题

## 5. 大陆与台湾差异

### 5.1 大陆

- 国家级规划和统计很强。
- 主要港口的政策与布局口径明确。
- 但统一点位主层弱，因此必须补几何。

### 5.2 台湾

- 官方港群和商港规划、统计也很强。
- 部分港口/码头点位开放条件比大陆更好。
- 但要做“台湾主要商港点层”仍然需要从规划与统计中筛选，再补统一点几何。

## 6. 与现有仓库架构的承接判断

- 适合直接做成 `ports` 点层
- 首版应明确写成 `主要商港/关键港口节点层`
- 构建期必须先从大陆与台湾的规划/统计里筛出主要节点，再补点位几何

## 7. 与日本最明显的不同

- 日本港口的问题更偏“旧但对题的专题源能否直接用”。
- 中国港口的问题更偏“规划和统计强，但统一点位主层弱”。
- 这使中国港口比日本更适合先做“关键节点层”，而不是试图复制一张完整港湾专题图。

## 8. 风险与下一步建议

1. 最大风险是把港口规划和吞吐量统计误写成现成点层。
2. 第二个风险是把所有港点都纳入首版，导致层失焦。
3. 第三个风险是忘了中国这组必须同时处理大陆与台湾，不应只研究沿海大陆港口。
4. 中国港口如果后续要开工，建议先只保留上海港、宁波舟山港、深圳港、广州港、青岛港、天津港、厦门港以及台湾港群中的关键商港节点。
