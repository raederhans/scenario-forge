# 中国工业区专题研究归档

日期：2026-03-28

## 1. 一句话结论

中国这条线是三条资源线里最适合先试点的：`mainland` 有国家级经开区、高新区等官方命名园区体系，但全国统一边界几何不足；`Taiwan` 则有官方 `臺灣各工業區範圍圖`。因此最稳的整体结论是 `官方命名园区体系成立，但 mainland 几何主源仍需补强；Taiwan 官方边界主源可用`。

## 2. 研究边界

- 只研究 `真实园区 / 工業用地`
- 不先用规划工业分区替代
- 优先接受“命名园区清单 + 实际工业用地代理层”的双层结构
- 本文默认同时比较 `mainland` 与 `Taiwan`

## 4. 主源 / 后备源 / 排除项

### 4.1 主源

- `mainland`：官方命名园区体系成立，主名单可由国家级经开区/高新区等官方名录承担，但边界几何主源缺口成立
- `Taiwan`：`臺灣各工業區範圍圖` 直接作为边界主源

### 4.2 后备源

- `mainland`：自然资源部开发区土地集约利用监测统计，用于确认“真实工业开发平台”而不是纸面规划区
- `Taiwan`：`國土利用監測計畫` 用于核对工业区开辟率和现况
- 跨区补边界：`OSM / Geofabrik`

### 4.3 排除项

- 纯规划工业分区
- 只讲招商或政策、不含真实园区对象的宣传名单
- 把城市工业用地统计直接等同于命名工业园区边界

## 5. 与现有仓库架构的承接判断

这条线最适合承接为 `industrial_zones` 面层资源包，而且中国包天然就是双层结构：

- `mainland`：`命名园区主名单 + 边界补强/工业用地代理`
- `Taiwan`：`官方边界主源 + 现况监测补强`

也就是说，这条线不应被做成点层。更稳的做法是保留园区面，必要时再为前端派生代表点。

## 6. 与日本最明显的不同

日本工业区试点更偏“全国真实工业用地专题层”；中国则更偏“双轨”：

- `mainland` 强在国家级命名园区体系
- `Taiwan` 强在官方边界图层

这意味着中国工业区不是一份单源数据包，而是最适合用“名单和边界分层”来做。

## 7. 风险与下一步建议

1. 最大风险是把 `mainland` 的国家级园区名单误写成“已经具备全国边界主源”。
2. 第二个风险是把所有工业用地都当成命名园区；这会违背当前任务边界。
3. `Taiwan` 这边虽然边界主源清楚，但园区类型更全，首版仍应优先保留工业区/产业园区主层，不把科学园区等所有园区一次性混入。
4. 建议首版优先试点这条线，因为它是三条资源线里最接近“官方命名体系明确、几何可补齐”的对象。
