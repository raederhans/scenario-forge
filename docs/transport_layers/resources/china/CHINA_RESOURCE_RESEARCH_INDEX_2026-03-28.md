# 中国工业矿产、能源设施、工业区研究总览

日期：2026-03-28

## 1. 总体结论

中国这三条线里，`工业区` 最适合先试点，`工业矿产` 可以做但 `mainland` 与 `Taiwan` 的官方主源形态差异很大，`能源设施` 则最依赖“mainland 官方统计 + Tier C 点位补全”的双层结构。整体上，中国比日本更碎，最稳的做法不是硬找一份全国统一主源，而是明确写出 `mainland / Taiwan` 的不同主源策略。

## 2. 三条线总体成熟度判断

| 线别 | 成熟度判断 | 现在能不能直接落盘 | 核心原因 |
|---|---|---|---|
| 工业区 | 中上 | 可以 | `mainland` 有官方命名园区体系，`Taiwan` 有官方工业区边界图；虽然不是单一主源，但已经足够构成可解释的双层主轴 |
| 工业矿产 | 中 | 可以，但要写清差异 | `mainland` 官方矿产地/地质资料体系存在但低摩擦主源较弱，`Taiwan` 的矿业资讯公开平台更直接 |
| 能源设施 | 中低 | 可以，但必须接受降级 | `mainland` 缺少顺手的全国官方点位主源，现实上要用官方统计锚点加 Tier C 电厂追踪器；`Taiwan` 官方电厂列表则较强 |

## 3. 哪条线最适合先试点

最适合先试点的是 `工业区`。

原因很直接：

1. 这条线虽然不是单源最整齐，但两边都能找到真实对象层，而不是只能拿统计表。
2. `mainland` 的国家级命名园区体系已经足够强，可以先做名单主轴。
3. `Taiwan` 的 `臺灣各工業區範圍圖` 直接提供了边界层。
4. 相比之下，能源设施在 mainland 的点位主源缺口更明显，工业矿产则更容易被资源分布和矿业权语义混淆。

## 4. 和日本最明显的差异

### 4.1 总体差异

日本现成专题包更像“全国单源可复用”；中国更像“官方体系存在，但 `mainland` 与 `Taiwan` 源头结构不同，必须分策略承接”。

### 4.2 工业矿产

日本矿产试点更接近全国资源点骨架；中国这条线一半像地质资料体系，一半像矿业权/矿区平台，源语义不统一。

### 4.3 能源设施

日本能源层主要是“全国锚点偏旧”；中国则是“mainland 统计强、点位弱，Taiwan 点位和电厂名单更直接”。

### 4.4 工业区

日本工业区更接近真实工业用地专题层；中国工业区更适合做成 `mainland 名单主轴 + Taiwan 边界主轴` 的双层结构。

## 5. 哪些地方必须降级到非官方但可信公开源

### 5.1 明确需要降级的地方

- `能源设施`：`mainland` 的全国点位主源缺口最明显，需要降级到 `Global Energy Monitor`、运营方公开资料等 Tier C 源。
- `工业区`：`mainland` 如果要求全国统一边界层，必须降级到 OSM 或地方园区门户补几何。
- `工业矿产`：`mainland` 如果低摩擦下载拿不到全国矿产地对象，也需要接受“官方体系存在但研究层落地要补 Tier C”的现实。

### 5.2 当前不必降级的地方

- `Taiwan` 的工业区边界主源不必降级。
- `Taiwan` 的能源设施主名单不必降级。
- `Taiwan` 的矿业信息主平台也不必先降级。

## 6. 可直接承接到仓库架构的判断

### 6.1 工业矿产

仍适合做 `mineral_resources`，但要接受 `mainland` 与 `Taiwan` 的几何和语义不完全一致；必要时从矿区或矿产地对象派生代表点。

### 6.2 能源设施

适合做 `energy_facilities` 点层，但首版应强行收敛到 `发电设施`，不要扩到油气管线、油库和 LNG 终端。

### 6.3 工业区

最适合做 `industrial_zones` 面层，而且应直接接受“名单和边界分层”的设计，不要强求一张单表解决所有问题。

## 7. 风险判断

1. 最大风险是把中国写成和日本一样的单源模式，这不成立。
2. 第二个风险是因为 `Taiwan` 某些源较清楚，就误以为 `mainland` 也同样低摩擦。
3. 第三个风险是把发展区、工业用地、矿业权、资源分布、发电设施这些不同语义偷换成一个“都能画”的对象层。

## 8. 下一步建议

1. 先把 `industrial_zones` 跑成中国资源线的首个试点。
2. `mineral_resources` 继续核对 `mainland` 官方矿产地体系到底能否低摩擦提取对象。
3. `energy_facilities` 在 `mainland` 先固定接受“官方统计 + Tier C 点位补全”的写法，不再等待一个未证实的全国官方点源出现。
