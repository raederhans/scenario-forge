# 中国铁路专题研究归档

日期：2026-03-28

## 1. 一句话结论

中国铁路这条线当前最稳的判断是 `OSM 主几何 + 官方属性加固`：中国大陆公开官方线几何主源不够顺，台湾站点开放资料明显更强，但整组口径下最可执行的方案仍然是让 OSM 扛线几何，再由大陆官方铁路信息和台湾官方站点资料补线路身份、站点和重要度。

## 2. 研究边界

- 研究对象固定为 `铁路线 + 主要车站`
- 不研究时间轴
- 不研究班次、routing、运行图
- 不研究全量车站设施系统
- 不研究城市轨道、地铁、轻轨的完整并表

## 4. 主源 / 后备源 / 排除项

### 4.1 主源

- `铁路线主几何`：OpenStreetMap / Geofabrik
- `台湾主要车站`：`臺鐵車站` + `臺鐵車站基本資料` + `各等級鐵路車站基本設施`
- `大陆主要车站和运营核对`：`铁路12306` + 国铁公开信息

### 4.2 后备源

- 国铁集团新闻、公报和年度统计
- 地方铁路运营方公开站点与线路页面
- 台湾个别运营单位公开站点页面

### 4.3 排除项

- 完整班次和运行图
- 全量地铁、轻轨、城际轨道
- 把国铁新闻稿、公报误写成现成 GIS 线层

## 5. 大陆与台湾差异

### 5.1 大陆

- 官方强项在于网络规模、公报、开通线路、客运站运营信息。
- 公开可直接批量拿到的铁路线几何主源弱。
- 因此大陆部分必须接受 `OSM 主几何 + 官方属性加固`。

### 5.2 台湾

- 官方开放站点资料明显更强，主要车站更容易做。
- 但就这轮“整组中国运输层”而言，台湾也没有比日本那样更干净的单一铁路线主包。
- 因此台湾部分虽然更接近 `官方主源 + OSM 补缺`，但整组仍按 `OSM 主几何 + 官方属性加固` 统一收口更稳。

## 6. 与现有仓库架构的承接判断

- 继续走 `railways` + `rail_stations` 两个 deferred context pack
- `railways` 统一以 OSM 为线主几何
- `rail_stations` 则允许大陆与台湾分别采用不同属性锚点
- 构建期应显式保留 `subregion = mainland / Taiwan`

## 7. 与日本最明显的不同

- 日本铁路更像“官方专题数据包先在”。
- 中国铁路更像“官方运营信息和站点信息强，但线几何主源要靠协作源”。
- 台湾比大陆更接近日本，但仍不足以把整组中国铁路直接写成 `官方主源 + OSM 补缺`。

## 8. 风险与下一步建议

1. 最大风险是把国铁公开信息误写成全国开放线几何主源。
2. 第二个风险是把台湾较强的站点开放条件外推到中国大陆。
3. 第三个风险是因为大陆主几何不齐，就拒绝形成结论；这轮应直接接受 `OSM 主几何 + 官方属性加固`。
4. 建议后续如果真要做中国铁路首版，优先先跑 `主要车站`，不要先冲全站。
