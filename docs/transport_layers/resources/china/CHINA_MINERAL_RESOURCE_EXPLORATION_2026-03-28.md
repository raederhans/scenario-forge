# 中国工业矿产专题研究归档

日期：2026-03-28

## 1. 一句话结论

中国这条线可以研究，但不能假装 `mainland` 与 `Taiwan` 共享同一套低摩擦主源：`mainland` 更像“官方矿产地/地质资料体系存在，但公开拿来即用的全国资源点主源摩擦较高”，`Taiwan` 则更接近“官方矿业信息平台 + 地质图资服务可直接用”；因此整条线最稳的写法是 `官方体系存在，但中国整体的一体化低摩擦主源缺口成立`。

## 2. 研究边界

- 研究对象是 `矿床 / 矿产地 / 资源分布`
- 首版优先接受 `点` 或 `代表性资源分布对象`
- 不强求反映当前是否在产
- 不承诺做矿权或采矿经营状态层
- 本文默认同时比较 `mainland` 与 `Taiwan`

## 4. 主源 / 后备源 / 排除项

### 4.1 主源

- `mainland`：以自然资源部数据服务体系与全国地质资料数据中心相关入口为制度主源，但研究层面要明确“低摩擦主源缺口成立”
- `Taiwan`：以 `礦業資訊公開平台` 为主源

### 4.2 后备源

- `mainland`：`中国矿产资源报告`、公开地学论文与专题数据只做属性和区域核对
- `Taiwan`：`地質資料整合查詢服務`
- 跨区补缺：`Mindat` 等公开可核验矿点目录，只能 Tier C 降级使用

### 4.3 排除项

- 现役矿山经营状态
- 矿业公司生产经营层
- 储量、产量和矿权交易动态
- 把矿业权直接等同于资源分布强度

## 5. 与现有仓库架构的承接判断

这条线仍适合落到 `mineral_resources`，但中国不适合被简单写成“全国纯点层”。更稳的承接方式是：

- `mainland`：优先接受“官方体系 + 派生代表点”的结构
- `Taiwan`：可直接以矿区/矿业权对象或代表点进入图层
- 前端如果只接点，构建期应允许从矿区面或矿区对象派生 `representative_points`

也就是说，中国这条线在仓库里仍能走独立 deferred pack，但构建端必须接受不同子区域的源语义并不完全一致。

## 6. 与日本最明显的不同

日本矿产试点更接近“全国资源点骨架已经直接存在”；中国这条线更像“mainland 有大体系但公开落地摩擦高，Taiwan 有更直接的矿业信息公开平台”。  

最关键的差异不是矿产资源少，而是：

- 日本主源更像直接可取的全国点集
- 中国 mainland 更像机构级数据中心与报告体系
- Taiwan 更像可直接查询的矿区/矿业权空间平台

## 7. 风险与下一步建议

1. 最大风险是把 `mainland` 的官方矿产体系误写成“已经存在一份可直接下载的全国资源点主表”。目前能确认的是官方体系存在，不是低摩擦产品一定存在。
2. 第二个风险是把 `Taiwan` 的矿业权/矿区对象直接等同于资源分布点集。
3. 如果后续产品只接受点，必须在构建期明确“由矿区对象派生代表点”的规则，不能在研究文档里偷换成天然点源。
4. 建议顺序是：先把 `Taiwan` 官方平台的字段和几何摸清，再确认 `mainland` 是否真能从国家地质资料体系顺利拿到矿产地对象；若不能，立即把 `mainland` 写成“官方低摩擦主源缺口成立”。
