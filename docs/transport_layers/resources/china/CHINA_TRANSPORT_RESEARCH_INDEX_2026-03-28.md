# 中国交通专题研究总览

日期：2026-03-28

## 1. 一句话结论

中国这四条线里，`机场` 最适合先试点，`铁路` 可以做但整体更适合 `OSM 主几何 + 官方属性加固`，`港口` 适合先收成 `主要商港/关键港口节点层`，`公路` 则最依赖 `OSM 主几何 + 官方路线方案/设施坐标补强`；台湾在点状设施和部分站点数据上明显强于中国大陆，但中国大陆在国家级统计、公报和路线规划上更强。

## 2. 四条线总体成熟度判断

| 线别 | 总体成熟度 | 大陆最强主锚点 | 台湾最强主锚点 | 是否存在全国统一官方主源 | 与日本最明显的不同 |
|---|---|---|---|---|---|
| 铁路 | 中 | `12306 / 国铁官方信息 + 国务院/国铁统计` | `data.gov.tw 台铁车站基本资料 / 各等級鐵路車站基本設施` | 中国整体不成立；台湾站点强于大陆 | 日本铁路更像“专题包先在”；中国更像“官方站点/统计强，但公开线几何弱” |
| 公路 | 中低 | `交通运输部国家公路网规划与路线编号` | `高速公路局交流道/门架/路径数据` | 中国整体不成立 | 日本公路还能围绕单国专题加固；中国整体更像“官方方案强、公开几何弱” |
| 机场 | 中上 | `民航局运输机场生产统计公报` | `民航局/政府资料开放平台机场点位与营运数据` | 台湾较强，大陆偏名单/统计，中国整体不成立 | 日本机场更像现成点层；中国整体更像“官方名录和统计 + 几何补充” |
| 港口 | 中 | `交通运输部主要港口规划与吞吐量统计` | `航港局/iMarine/台湾港务公司港群资料` | 中国整体不成立 | 日本港口有旧但对题的专题包；中国整体更像“规划和统计强、点位主层弱” |

当前排序：`机场 > 铁路 > 港口 > 公路`

## 3. 哪条线最适合先试点

当前最适合先试点的是 `机场`。

原因很直接：

1. 机场本来就是点图层，密度可控。
2. 中国大陆和台湾两边都能找到官方或准官方机场名单、吞吐量或营运数据。
3. 台湾一侧已经有政府开放平台机场点位类数据；大陆虽然点位主包弱，但对象范围和重要度口径很清楚。
4. 相比铁路和公路，机场更不依赖一张完美的全国线网几何主源。

## 4. 和日本最明显的不同

和日本相比，中国最明显的不同不是“完全没有官方源”，而是 `官方公开信息更偏统计、公报、规划和设施目录，直接可产品化的专题 GIS 主包明显更少`。

- 中国大陆更强的是国家级统计、公报、路线规划、行业名录。
- 台湾更强的是政府资料开放平台上的点位和设施数据。
- 所以中国这组不能像日本那样默认期待“一份官方全国专题包”解决问题，而要从一开始就接受大陆与台湾的信息形态不同。

## 5. 哪些地方必须降级到非官方但可信公开源

### 5.1 铁路线几何

- 中国大陆公开官方线几何主源不够顺。
- 台湾站点开放资料较强，但公开可直接承接的完整铁路线主包也不如日本整齐。
- 因此中国铁路整体应接受 `OSM / Geofabrik` 作为首版主几何。

### 5.2 公路线几何

- 中国大陆的国家公路网方案、路线编号和高速公路建设进展都很强，但公开静态全国主干路几何主源仍然不稳。
- 台湾高速公路局公开设施点、路径和路况数据较多，但静态全国主干路主源也不是 UK/Germany 那样一张干净总表。
- 因此中国公路整体也应接受 `OSM / Geofabrik` 为主几何。

### 5.3 机场与港口点位

- 大陆机场、港口的官方名录和统计强于点位主层。
- 台湾在机场和部分港口点位上更好，但仍不等于中国整体已经有一套统一官方点层。
- 因此中国机场与港口都要接受 `官方名单/统计 + 点位补充` 的现实。

## 6. 与现有仓库架构的承接判断

- `railways`
  - 适合独立 deferred context pack
  - 线层整体按 `OSM 主几何 + 官方属性加固`
  - 主要车站单独做点层
- `roads`
  - 适合 `roads` + `road_labels`
  - 首版仍只做 `motorway / trunk / primary`
- `airports`
  - 最适合复用现有 `cityPoints` 风格的设施点层承接链
  - 中国这条线最先可落
- `ports`
  - 也适合点图层承接
  - 但必须明确写成 `主要商港/关键港口节点层`

## 7. 风险与下一步建议

1. 最大风险是把大陆与台湾硬写成一套同质数据条件，这不成立。
2. 第二个风险是把规划、公报和统计误写成已经具备 GIS 主层。
3. 第三个风险是因为大陆点位源不齐，就无限推迟写文档；这轮不应该这样做，应直接写成缺口成立或降级使用。
4. 建议首版顺序：
   1. `机场`
   2. `铁路`
   3. `港口`
   4. `公路`
