# 日本公路试点探索归档
日期：2026-03-28

## 1. 目标

这份文档单独归档 `日本公路图层试点` 的探索结果，用来锁定首版试点的范围、数据源策略和架构接法。

本次讨论后的结论已经固定为：

- 只做 `最新快照`
- 不做历史回溯
- 采用 `功能等级路网`
- 日本试点先做 `高速 + trunk + primary`
- 用 `OSM 主几何 + 官方高速加固`

## 2. 为什么选日本

日本适合做公路试点，不是因为它最简单，而是因为它能尽早把真实问题暴露出来：

1. 都市圈密度高，最能检验图层是否会失控
2. 高速网络有较好的官方开放数据
3. 全国道路主骨架又不能完全依赖官方开放源
4. 很适合验证“OSM 主几何 + 官方属性加固”这条路线是否成立

如果日本公路试点能跑通，后续复制到其他国家会更稳。

## 3. 已确认的产品边界

### 3.1 做什么

- 日本全国道路包
- 线图层
- 编号图层
- 项目保存与恢复
- 重点验收关东与关西高密区域

### 3.2 不做什么

- 不做历史时间轴
- 不做互通点
- 不做 routing
- 不做全量普通道路
- 不做 secondary 及以下
- 不做复杂节点体系

## 4. 数据源探索结论

## 4.1 总结

日本公路试点不能照搬铁路试点的“官方主源 + OSM 补缺”。

原因：

- 日本官方开放数据里，`高速道路` 的专题源很好
- 但 `当前全国主干公路网` 并没有一个同样成熟、免费、开放、即拿即用的官方主源
- 真正强的全国道路库，如 DRM / xROAD，接入门槛和许可都不适合这次开放试点

所以首版应明确承认：

- `OSM / Geofabrik Japan` 是当前全国主干路网的几何主源
- 官方高速数据只承担“高速身份和属性校正”的职责

## 4.2 关键来源判断

### 适合进入首版主链路

- [Geofabrik Japan](https://download.geofabrik.de/asia/japan.html)
  - 作为 OSM 日本提取的工程入口
  - 负责当前全国道路线主骨架
- [国土数値情報 N06 高速道路時系列](https://nlftp.mlit.go.jp/ksj/gml/datalist/KsjTmplt-N06-v2_0.html)
  - 负责高速公路官方身份、名称、编号等加固

### 适合做后续增强，不进首版主链路

- [国土数値情報 N10 緊急輸送道路](https://nlftp.mlit.go.jp/ksj/gml/datalist/KsjTmplt-N10-2024.html)
  - 适合以后做战略道路专题层
- [国土数値情報 N01 道路](https://nlftp.mlit.go.jp/ksj/gmlold/datalist/gmlold_KsjTmplt-N01.html)
  - 太旧，不适合做现代当前主图层

### 明确不作为本次开放试点主源

- [DRM 数据库](https://www.drm.jp/database/content/)
- [xROAD 道路基盘地図等数据库](https://www.xroad.mlit.go.jp/database/430/)

原因不是它们不好，而是：

- 许可与接入门槛更高
- 不适合作为这次开放试点的最短路径

## 5. 日本试点的数据策略

### 5.1 主源策略

固定为：

- `OSM 主几何`
- `官方高速加固`

具体含义：

- 从 OSM 中抽取日本全国 `motorway/trunk/primary`
- 用 N06 对 `motorway` 做官方校正
- 不让官方高速数据反向主导全国普通干路几何

### 5.2 首版收录下限

首版只收：

- `motorway`
- `trunk`
- `primary`

首版默认排除：

- `motorway_link`
- `trunk_link`
- `primary_link`
- `service`
- `residential`
- `track`

### 5.3 编号策略

首版编号只做：

- 高速编号
- 国道主编号

首版不做：

- 所有道路尽量编号
- 复杂的多体系道路编号并列展示

## 6. 架构探索结论

根据当前仓库，最合适的接法不是改主拓扑，而是沿用现有 context layer 管线。

固定结论：

- 新建独立 `roads` context pack
- 新建独立 `road_labels` pack
- 默认懒加载
- UI 和项目持久化接入现有 context layer 流程

这条路线和铁路类似，但公路比铁路多出三个必须提前落地的约束：

1. 必须有严格等级过滤
2. 必须有按缩放分层显示
3. 必须有首版降噪规则

## 7. 日本试点的首版技术口径

### 7.1 内部统一 schema

建议固定为：

| 字段 | 含义 |
|---|---|
| `id` | 稳定主键 |
| `name` | 道路名称 |
| `ref` | 道路编号 |
| `road_class` | `motorway/trunk/primary` |
| `source` | `osm_jp` / `official_n06` |
| `official_ref` | 官方高速编号，可选 |
| `geometry` | 线几何 |

### 7.2 缩放规则

- 低缩放：只显示 `motorway + trunk`
- 中缩放：增加 `primary`
- 高缩放：增加更密的编号显示

首版仍然不开放次级道路。

### 7.3 降噪规则

首版必须做的降噪：

- 排除 link/service 类支线
- 对连续同类线段做安全合并
- 对非常短的投影线段设置最小显示阈值

首版明确不做：

- 激进双车道折叠
- 模糊启发式聚合
- 事后补丁式修图

## 8. 试点实施顺序

建议按这个顺序推进：

1. 先构建全国 `roads` pack
2. 单独构建 `road_labels` pack
3. 接入懒加载和图层开关
4. 接入缩放分层与样式配置
5. 接入编号显示
6. 在东京圈和大阪圈做高密场景验收
7. 再抽验一处低密区域确认全国一致性

## 9. 一句话结论

日本公路试点的正确起点是：

`用 OSM 做日本全国主干路网几何骨架，用官方 N06 只加固高速身份和编号，再把图层作为独立 deferred context pack 接进现有架构，同时把分级显示和降噪放进首版。`
