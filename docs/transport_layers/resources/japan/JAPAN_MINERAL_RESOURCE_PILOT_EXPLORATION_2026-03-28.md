# 日本工业矿产试点探索归档

日期：2026-03-28

## 1. 一句话结论

如果日本这条线的目标是 `矿床/资源分布`，而不是“现役矿山”，那么 GSJ 的 [国内の鉱床・鉱徴地に関する位置データ集（第2版）](https://www.gsj.jp/publications/pub/prompt-rep/index.html) 已经足够构成一个可用的全国资源点骨架。

## 2. 研究边界

本轮固定边界如下：

- 研究对象是 `矿床 / 矿徴地 / 资源分布`
- 首版几何是 `点`
- 不要求反映当前是否在产
- 不承诺做资源区面
- 不把矿山经营状态、产量、储量波动并进来

## 4. 为什么主源应该选 GSJ 第 2 版

主因不是“它最新”，而是它最贴题。

这套数据的优势在于：

- 目标直接就是 `概观国内矿物资源及燃料资源分布`
- 语义本身就是点位级的资源分布
- 来自 GSJ 这一条权威地学链路
- 列表页已经明确给出数据附件入口，说明它不是只有论文说明，没有数据

这一点比“自己从新闻、企业名录、现役矿山名单拼接”稳得多。

## 5. 日本试点主源 / 后备源 / 排除项

### 5.1 主源

- GSJ `国内の鉱床・鉱徴地に関する位置データ集（第2版）`

### 5.2 后备增强

- GSJ 2015 旧版位置数据集，用于比对字段与旧条目
- GSJ 地质信息数据库与地图服务，用于后续核点、补图幅出处、增强矿种解释

### 5.3 当前排除

- 现役矿山经营状态
- 产量、储量、采选能力
- 矿业公司经营主体层
- 资源区面或矿权范围面

## 6. 与现有仓库架构的承接判断

这条线非常适合先走点图层。

原因：

- 数据天然是资源点
- 没有必要强行构造线或面
- 可以直接复用 `cityPoints` 风格的点位渲染、标签密度、开关与项目持久化

因此首版承接建议固定为：

- `mineral_resources` 独立 deferred context layer pack
- 点图层懒加载
- 重要度、矿种、资源大类做前端筛选

## 7. 当前最该提前写清的风险

- 这层很容易被误读成“日本现役矿山分布图”，但它本质上更接近“资源点骨架图”。
- 不同图幅的矿种命名和图例表述可能不完全统一，字段规范化会是第一项真实工作。
- GSJ 发布页能确认数据附件存在，但如果以后用途从内部研究转向外部传播，仍应逐条核对该数据的复用边界。

## 8. 下一步建议

如果继续推进日本试点，这条线的最短路径是：

1. 先下载 GSJ 第 2 版数据附件，确认字段结构和坐标
2. 抽样核对北海道、东北、九州三类矿产带，判断是否能形成全国骨架
3. 先做 `resource_type` 和 `resource_class` 两层最小映射
4. 不额外承诺“在产状态”，把产品名称固定成 `资源分布`

## 9. 试点判断

在这三条专题层里，`工业矿产` 是一个适合较早进入产品试点的对象。

原因不是它最贴近现实工业活动，而是：

- 主源权威性强
- 点层承接简单
- 资源分布的主题边界清楚
- 不需要先解决复杂的网络或范围表达问题
