## 3. 日本部分已经收口的结论

### 3.1 交通层

#### 铁路

- 日本铁路是当前最完整、最清楚的官方试点之一。
- 当前口径已经锁定为：`官方主源 + OSM 补缺`。
- 首版重点不是全站点，而是 `线路 + 主要车站`。
- 历史维度不做连续时间轴，只保留 `active / disused / abandoned / construction` 这类状态边界。

#### 公路

- 日本公路和铁路不一样，现代全国当前路网更适合 `OSM 几何主源 + 官方属性加固`。
- 如果只做高速和骨干道路，官方源价值更高。
- 如果做全国当前主路网，必须更早处理缩放分级和降噪。
- 当前最稳的首版口径仍然是 `expressway + trunk + primary`，不先碰更低等级道路。

#### 机场与港口

- 机场和港口已经明确拆成 `设施本体`，不和航线、航路混做一层。
- 日本机场主源明显强于日本港口。
- 日本机场更适合先行，港口更适合并行研究后再决定是否进入实现。
- 这两条线在仓库里都更适合走 `点图层`，而不是复用线网思路。

### 3.2 专题层

#### 工业矿产

- 日本工业矿产当前最稳的定义是 `矿床 / 资源分布`。
- GSJ 的矿床与矿徴地位置数据已经足够做全国资源点骨架。
- 这条线的风险不在几何，而在语义误读，必须明确它不是“现役矿山层”。

#### 能源设施

- 日本能源设施当前最稳的全国锚点是 `P03 発電所データ`。
- 这意味着首版最可信的是“发电设施锚点层”，不是完整能源设施大全。
- 如果后续继续扩炼油厂、LNG 基地、油库、变电站，需要单独建立子专题判断，而不是现在硬并。

#### 工业区

- 日本工业区当前最贴题的主源是 `L05 工業用地データ`。
- `A29 用途地域` 只能做比较层和代理备选，不能替代真实工业用地。
- 在三条专题层里，日本工业区目前是最适合优先试点的对象。

## 4. 日本研究沉淀出来的可复用架构

### 4.1 三种承接方式

- `线图层`：铁路、公路，优先走独立 deferred context pack
- `点图层`：机场、港口、工业矿产、能源设施，优先复用 `cityPoints` 风格链路
- `面图层`：工业区，优先走 polygon/context layer

### 4.2 当前已经明确的产品边界

- 不在首版里追求全量设施名录
- 不在首版里引入连续时间轴
- 不用规划代理层偷换真实对象
- 不把网络层和设施层强行混在一起

### 4.3 对后续跨国复制最有价值的部分

- 统一研究模板已经成型
- 点、线、面三种对象的承接方式已经区分清楚
- “官方优先、必要时降级但显式标注”的规则已经明确
- 日本已经提供了一个足够完整的对照组，可以拿来比较德国、英国、法国、美国的缺口

## 5. 日本作为后续四国扩展的模板入口

后续德国、英国、法国、美国的研究，应当直接复用日本已经锁定的三条专题口径：

- `工业矿产` = `矿床/资源分布`
- `能源设施` = `点状设施`
- `工业区` = `真实园区 / 工業用地`

同时直接复用日本已经沉淀出的文档结构：

- 1 份国家总览
- 3 份专题探索
- 每份专题都要有数据源判断表、主源判断、架构承接判断、风险与下一步建议

## 6. 日本部分的当前优先顺序

如果只看已经完成的日本研究，当前更适合往实现方向靠拢的顺序是：

1. `工业区`
2. `工业矿产`
3. `能源设施`

如果回到交通层，则当前最清楚的顺序仍然是：

1. `铁路`
2. `高速 / 骨干公路`
3. `机场`
4. `港口`

## 7. 一句话结论

日本部分现在已经不再是零散的单篇研究，而是一个可以直接拿来复制的模板集合：

`方法论留在 methodology，国家研究留在 resources/japan；后续四国扩展只需要沿用同一研究模板，重点比较哪里和日本一样，哪里不得不降级。`
