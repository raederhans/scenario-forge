# App/Web 汉化执行计划

## 范围
- 检查主 app shell、landing 页面、HTML/JS 中用户可见英文文案。
- 更新现有汉化脚本或 locale 数据，优先扩展现有文件。
- 补充或更新现有 i18n/source-level contract 测试。

## 步骤
- [ ] 扫描现有 i18n 资源、汉化脚本和可见英文文案。
- [ ] 区分用户可见文案、代码标识符、第三方库、非可见脚本内容。
- [ ] 最小改动汉化真实漏点。
- [ ] 更新现有测试或脚本覆盖漏点。
- [ ] 运行定向验证。
- [ ] 复核 diff，完成 review 查 bug。

## 完成标准
- 用户可见主 app / landing 文案没有明显英文漏点。
- 汉化脚本或 locale 源能覆盖本次新增/修复项。
- 定向 i18n 测试通过。
