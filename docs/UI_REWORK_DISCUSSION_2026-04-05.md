## 已达成决议
- 主界面的第一原则是 `地图优先`，地图必须始终是第一视觉主角。
- 主要用户按 `模组作者 / 玩家` 设计，不按纯专业绘图师设计。
- 产品整体气质定为 `专业克制 + 轻策略感`。
- 右栏是唯一主工作栏，复杂编辑必须收束到这里。
- 左栏采用 `窄模式栏 + 可展开文字层`，主要承载 `模式 + 一级任务入口`。
- 左栏不承担深层参数编辑，不作为第二主工作栏。
- 顶部 `scenario context bar` 只保留为极轻状态条，不再承担主编辑动作。
- `bottom dock` 默认收起，只在特定任务里出现。
- `bottom dock` 只承载当前模式的短动作，以及极少数通用短动作，如撤销、重做。
- dock 不是第二工作台，不承载需要阅读、配置或管理的内容。
- 右栏不再按 `Inspector / Project` 的平级栏目组织，而是按同一工作栏中的不同 `scope` 组织。
- `scope` 切换策略定为 `自动默认 + 手动锁定`。
- 手动锁定会持续到用户离开当前任务为止。
- 空选择状态下，右栏默认落在 `项目 scope`。
- 引导采用双层结构：
  - 左栏负责解释当前模式是什么。
  - 右栏负责解释当前任务正在做什么。
- 浮层系统只负责短时反馈与局部补充，不承担长期工作区职责。
- 启动 overlay 不再占据首屏中心位置。

## 被否决或受限方向
- 不把“把更多面板压成 Adobe 式工具条”作为第一解。
- 可以借 `工具带的占地效率`，但不借 `把复杂编辑图标化` 的工作模型。
- 不保留 `Inspector / Project` 作为右栏中的平级双 tab 主关系。
- 不保留 `bottom dock` 作为综合工具区或第二工作台。
- 不允许 overlay、popover、hint 继续承担长期悬挂的半工作区职责。
- 不采用 `纯自动` 的 `scope` 切换策略。
- 不采用 `纯手动` 的 `scope` 切换策略。
- 不采用 `每次新选择都重置` 的手动锁定生命周期。
- 不把左栏继续扩张成缩小版主工作栏。
- 不把新手引导完全压进启动 overlay、顶部状态条或单一左栏说明中。

## 仍待继续讨论
- 字体序列与标题层级：
  - 标题、正文、标签、kicker、胶囊文字应该压成几层。
  - 哪些层级必须合并，哪些层级必须拉开。
  - 轻策略感应该通过哪些 typographic signals 进入，而不破坏可读性。
- 按钮语法与控件层级：
  - 主动作、次动作、模式切换、危险动作如何形成稳定语法。
  - 哪些动作应该继续留在右栏，哪些只适合短动作层。
- 信息密度与留白节奏：
  - 地图、侧栏、状态条之间的节奏如何重新分配。
  - 什么叫“密度高但顺序清楚”，什么叫“密度高且重心发散”。
- 状态表达与恢复：
  - 哪些 UI 状态必须是显性的。
  - 哪些状态后续需要被 URL 或会话恢复机制承载。

## 讨论记录

### 2026-04-05 第一轮：结构骨架
- 先确认“地图优先”而不是“编辑台优先”。
- 用户提出希望借 Adobe 系列软件的工具条感，为地图让出更多视觉权重。
- 讨论后的结论不是直接把更多面板工具条化，而是只借工具带的轻量占地，不把复杂编辑内容图标化。
- 先后定下：
  - 右栏是唯一主工作栏
  - 左栏改为窄模式栏
  - 顶部条改为轻状态条
  - dock 默认收起
  - 浮层退回短时反馈
- 在右栏组织方式上，确认 `项目级` 和 `对象级` 内容不是互斥关系，但也不应该被表达为平级双 tab，而应该被表达为同一条工作栏中的不同 `scope`。
- 在引导结构上，确认单靠左栏说明不够，因此采用“双层引导”：左栏解释模式，右栏解释当前任务。
